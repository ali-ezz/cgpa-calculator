# Contributing

Thank you for contributing to this repository.

## How to contribute

1. Fork the repository or branch from the main repository.
2. Create a new branch with a descriptive name, for example:
   - `fix/gpa-calculation-bug`
   - `feat/rtl-accessibility`
3. Make your changes and commit them with clear messages.
4. Open a pull request against `main`.

## Issue reports

- Use the bug report template for reproducible problems.
- Use the feature request template for desired enhancements.

## Pull request expectations

- Include a short summary of the change.
- Explain why the change is needed.
- Include testing steps or verification details.
- Keep PRs focused and small when possible.

## Branch naming

- `fix/...` for bug fixes
- `feat/...` for new features
- `docs/...` for documentation updates
- `chore/...` for maintenance tasks

## Review

Pull requests should be reviewed by maintainers or code owners before merging.
